Isinya seperti ini: menuentry "Fedora 22 ISO" --class fedora { set isoname="Fedora-Live-Workstation-i686-22-3.iso" set isopath="/home/poros/Downloads/" set isofile="${isopath}/${isoname}" echo "Using ${isoname}..." loopback loop (hd0,8)$isofile linux (loop)/isolinux/vmlinuz0 iso-scan/filename=${isofile} root=live:CDLABEL=Fedora-Live-WS-i686-22-3 rootfstype=auto ro rd.live.image quiet rhgb rd.luks=0 rd.md=0 rd.dm=0 initrd (loop)/isolinux/initrd0.img } Ubah nilai isoname sesuai dengan nama ISO Fedora yang kamu punya, di sini saya punya Fedora-Live-Workstation-i686-22-3.

Mengatasi Error Debootstrap di Fedora 22: E: Error executing gpgv to check Release signature

@alunux · Sep 4, 2015 · 1 min read

Bismillahirrahmanirrahim. Beberapa waktu yang lalu saya membuat sebuah container sederhana menggunakan systemd-nspawn untuk keperluan nyoba-nyoba aplikasi di debian/ubuntu, salah satunya alldeb. Ketika saya menjalankan debootstrap, saya mendapatkan pesan error sebagai berikut. Pesan error: Error executing gpgv to check Release signature. Setelah dicek, ternyata software bernama gpgv belum terpasang (adanya gpgv2). gpgv tersedia dalam paket gnupg. Jalankan “sudo dnf install gnupg” untuk mendapatkan gpgv. Setelah instalasi selesai, jalankan kembali perintah debootstrap.

Bismillahirrahmanirrahim. Alhamdulillah, salah satu proyek liburan saya telah rampung sekitar 57%. Proyek tersebut bernama ONERPM. Apa itu ONERPM? ONERPM adalah sebuah bash script sederhana berbasis DNF yang berguna untuk memudahkan pemasangan sebuah paket aplikasi di Fedora secara offline. Proyek ONERPM ini terinspirasi dari proyek alldeb buatan Nifa Dwi Kurniawan dan Fedora Offline Package Manager.
