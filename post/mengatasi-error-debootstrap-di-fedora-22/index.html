Mengatasi Error Debootstrap di Fedora 22: E: Error executing gpgv to check Release signature

@alunux · Sep 4, 2015 · 1 min read

Bismillahirrahmanirrahim.

Beberapa waktu yang lalu saya membuat sebuah container sederhana menggunakan systemd-nspawn untuk keperluan nyoba-nyoba aplikasi di debian/ubuntu, salah satunya alldeb. Ketika saya menjalankan debootstrap, saya mendapatkan pesan error sebagai berikut.

Pesan error:  Error executing gpgv to check Release signature. Setelah dicek, ternyata software bernama gpgv belum terpasang (adanya gpgv2). gpgv tersedia dalam paket gnupg. Jalankan “sudo dnf install gnupg” untuk mendapatkan gpgv. Setelah instalasi selesai, jalankan kembali perintah debootstrap.

Tulisan ini dikerjakan menggunakan WordPress dengan bantuan Google Chrome 45.0.2454.85. Sistem operasi yang digunakan yaitu Fedora 22. Shell menggunakan Bash dengan powerline-shell dijalankan menggunakan GNOME Terminal dengan font Meslo for Powerline. Semua screenshot yang ada di tulisan ini diambil dengan fitur screenshot GNOME.
